# 1. 产品概述

## 1.1 一句话定义

一款运行在 1.8 英寸 AMOLED 触屏开发板上的**像素风桌面生态缸**，主打「**低操作 · 长陪伴 · 高颜值**」。玩家不再"抽卡"，而是被动接收来自海鸥、漂流瓶、海风、客人等带来的随机礼物，养出一缸属于自己的微型世界。

## 1.2 设计原则

- **治愈优先**：操作不超过 3 步、永不死亡（最多睡眠/休眠）、配色柔和。
- **生态自洽**：缸内有真实的食物链与生长循环，鱼会变大、藻类会繁殖。
- **失衡即剧情**：参数偏离会触发剧情事件，但绝不变成失败惩罚。
- **硬件即玩法**：屏 / 麦 / IMU / RTC / 电池 全部参与到游戏循环。
- **桌面常驻**：不玩时自动变成时钟挂件，永远是桌面 C 位。

## 1.3 目标用户

- 桌面摆件党 / 解压玩家
- ESP32 / 极客折腾爱好者
- 喜欢 idle、放置、养成类手游的轻度玩家

## 1.4 核心循环（Core Loop）

<callout emoji="🔁">
登录看缸 → 收离线产出 → 喂食 / 互动 → **遭遇随机事件得到生物或礼物** → 装饰升级 → 截图打卡 → 离线
</callout>

# 3. 玩法系统设计

## 3.1 主线：放置养成

每只生物有 4 维状态：

| 属性 | 范围 | 规则 |
|-|-|-|
| 饱食度 | 0–100 | 每小时 -5；进食 +20\~50 |
| 心情 | 0–100 | 与饱食度联动；互动 +10 |
| 活力 | 0–100 | 白天上升、夜晚下降；电量 < 20% 时减半 |
| 体型 size | 0–100 | 累计进食触发成长，详见 §4 |

## 3.2 离线产出

累积"**光合币 ☀**"，公式：`光合币 = 在线生物数 × 系数 × 时长(h)`。回来时弹出"☀ 离线收益 +128"小动画，上限按缸等级递增。

## 3.3 互动方式

- **点击**：撒鱼食 / 摸头
- **吹气**：水面涟漪 + 蒲公英飘散
- **摇一摇**：水波 + 生物受惊（轻摇愉悦、重摇扣心情）
- **倾斜**：水位倾斜，鱼游向低处

## 3.4 商店与升级

- **装饰物**：城堡 / 假山 / 漂浮木 / 像素水草（可拖拽）
- **缸体皮肤**：圆缸 / 方缸 / 球缸 / 复古绿屏缸
- **缸等级**：升级提升离线收益上限、容纳生物上限
- **画笔解锁**：玻璃壁可手绘装饰，解锁更多颜色

<callout emoji="📌">
商店**不出售生物**。所有生物只能通过随机事件、繁殖、计步彩蛋获得，强化"自然相遇"的治愈感。
</callout>

## 3.5 桌面伴侣模式

不操作 60 秒后自动进入 Ambient Mode：

- 左下角时间，右上角电量
- 缸内生态自动循环演出
- AMOLED 黑色像素不发光，极省电
- 单击屏幕任意位置返回游戏

## 3.6 季节与节气

跟随 RTC 真实时间：春樱 / 夏荷 / 秋枫 / 冬雪 四季缸景自动切换；24 节气推送限定生物 / 装饰；主人生日触发蛋糕彩蛋。

# 4. 生态系统运行逻辑

<callout emoji="🌱">
方案 A 的生态系统是"**自带稳态修复的弹性生态**"——失衡只是剧情，不是惩罚。
</callout>

## 4.1 生态层级

```text
[环境层] 阳光 ☀ + 营养 💧（不可视，状态条显示）
   ↓
[L1 生产者] 藻类 / 苔藓 / 水草    ← 光合 → 产氧 + 涨"生物量"
   ↓
[L2 食藻者] 小虾 / 蜗牛 / 蝌蚪     ← 啃食 L1
   ↓
[L3 食肉者] 小鱼                  ← 捕食 L2 + 偶尔 L1
   ↓
[L4 顶级]   大鱼 / 异种 / 稀有     ← 捕食 L3，偶尔 L2
```

每层之间是"**主食 + 备选食物**"，备选效率低但能避免饿死。

## 4.2 心跳循环（每秒一帧）

整个生态以 **1 Hz 心跳** 推进，每一拍按顺序执行：

1. 环境更新（阳光、营养、氧气）
2. L1 生长（藻类/水草生物量）
3. L2 / L3 / L4 觅食（AI 找最近可食目标）
4. 个体成长（吃饱 → 长大）
5. 繁殖产卵（条件达成 → 漂出蛋）
6. 衰减/睡眠（饥饿超时 → 沉底）
7. 死亡分解（睡眠超 24h → 化为营养）
8. 玩家事件回收（喂食/抚摸）

## 4.3 环境层

| 参数 | 范围 | 增长 | 衰减 |
|-|-|-|-|
| 阳光 ☀ | 0–100 | 白天 +1/min（电量充足） | 夜晚 -0.5/min |
| 营养 💧 | 0–100 | 死亡分解、玩家投肥 | 被 L1 消耗 -0.2/min·棵 |
| 氧气 💨 | 0–100 | L1 光合产氧 | L2/L3/L4 呼吸消耗 |

## 4.4 L1 生产者（藻类/水草）

生长公式（Logistic 曲线，避免爆缸）：

```text
ΔBiomass = k1 × 阳光 × 营养 × (1 - Biomass / Capacity)
```

**柔性藻类爆发**：营养 > 80 持续 1 小时 → 屏幕变绿色滤镜，玩家点屏幕"擦缸"3 次即可清理。**不会真死鱼**。

## 4.5 L2 食藻者

- hunger 每分钟 +0.5
- 觅食优先 L1 藻类，次选浮游营养
- 进食时每秒 hunger -2，藻类生物量 -0.1
- 连续 30 分钟饱食（hunger < 30）→ growth_pts +1

## 4.6 L3 食肉者（小鱼）

捕食 AI：

1. 寻找半径 R 内体型 ≤ 自己 0.7 倍 的 L2/L3 个体
2. 若有 → 加速接近 → 接触判定 → 进食
3. 若无 → 在缸内悠游（贝塞尔曲线随机路径）

**大鱼吃小鱼**：允许小鱼吃同类，但只有体型差 ≥ 0.5 倍才发生。吃饱后 30s 内不再捕食（动画上"打嗝"）。

## 4.7 L4 顶级（大鱼）

- 缸里最多存在 2 条（防止灭绝）
- 每天至少需要 1 次进食，否则进入睡眠态
- 体型上限较高，吃饱时游动威严缓慢
- 无发光描边特效（v0.2 移除）

## 4.8 个体成长（鱼会变大）

| 字段 | 说明 |
|-|-|
| size | 0–100，像素显示按 size/100 缩放，最低 0.4 倍 |
| growth_pts | 成长点，进食累积 |
| stage | 幼体 / 亚成体 / 成体 / 巨型，对应 4 套 sprite |

每次进食 → `growth_pts += 食物营养值 × 个体物种系数`；达阈值 → `size +5`，触发"长大"小动画 + 音效。**玩家会真切看到自己的鱼从一个像素小点变成一条威武大鱼**。

## 4.9 繁殖产卵（克制）

- 条件：两条同种成体 + 饱食度 > 70 + 缸内同种数 < 上限
- 每日最多 1 次，产 1 颗"野生蛋"漂在水面
- 玩家点蛋 → 进入图鉴解锁流程（无稀有度概念，仅"已收藏 / 未收藏"）

## 4.10 死亡与回收（柔性）

- hunger > 95 持续 30 分钟 → 进入"睡眠态"（沉底像素静止 + 鼾泡）
- 玩家可点击睡眠生物喂食 → 立即唤醒
- 睡眠超 24h 无人理会 → 温柔消失（变成像素气泡飘走）+ 营养 +20
- **没有血淋淋的死亡画面**

## 4.11 稳态修复（治愈精髓）

| 现象 | 自动修复 |
|-|-|
| 藻类爆发 | L2 啃食加速 + 屏幕提示玩家擦缸 |
| 鱼太多 | 觅食半径自动放大、生育自然停止 |
| 鱼太少 | 离线时小概率"野生访客"漂入 |
| 营养枯竭 | 玩家充电 → 触发"营养雨" |
| 阳光不足 | 自动开缸内灯（消耗微量光合币） |

# 5. 随机事件系统

<callout emoji="💡">
v0.2 用**随机事件**取代抽卡。事件由真实世界状态触发（时间 / 步数 / 天气 / 充电），让"获得新生物"成为一种**自然相遇**。
</callout>

## 5.1 事件类型

| 事件 | 触发条件 | 奖励 |
|-|-|-|
| 🕊️ 海鸥来访 | 每日随机 1 次（白天时段） | 从天而降一个礼物盒，随机生物 / 装饰 / 光合币 |
| 🍾 漂流瓶 | 缸闲置 4h 以上回到主界面 | 瓶里有手写信 + 1 颗野生蛋 |
| 🌬️ 海风带来 | 计步达 1000 步 | 蒲公英/小昆虫漂入缸内 |
| 🌧️ 下雨日 | Wi-Fi 拉取真实天气=雨 | 营养 +30，限定青蛙/蝌蚪客人 |
| ⛈️ 雷暴日 | 真实天气=雷暴 | 限定生物（如电鳗），仅当日有效 |
| ❄️ 雪天 | 真实天气=雪 | 限定冬季生物（如冰晶虾） |
| 🌟 流星雨 | RTC 触发（每月固定 1 天） | 夜空特效 + 流星残骸（神秘装饰） |
| 🎋 节气访客 | RTC 节气日（24 节气） | 限定生物 / 装饰 |
| 🎂 生日彩蛋 | 主人生日（设置中预设） | 蛋糕装饰 + 限定生物 |
| 🔌 营养雨 | 插入 USB 充电 | 缸内下"营养雨"，加速 L1 生长 |
| 🌑 满月之夜 | RTC 农历计算 | 夜行物种活跃度 +50% |
| 🚪 客人造访 | 缸生态稳定 7 天 | 稀有访客（不入缸，停留 24h，喂食后留下纪念） |

## 5.2 事件呈现

1. **预告**：屏幕顶部出现小图标（海鸥剪影飞入）+ 提示音
2. **互动**：玩家点击事件图标 → 弹出事件卡片（如"海鸥带来了礼物"）
3. **开启**：点击"打开"按钮 → 礼物盒动画 → 显示获得物
4. **入缸/收藏**：生物自动入缸；装饰进背包；光合币直接入账

## 5.3 事件礼物池设计

事件礼物分 3 档（**无 SSR、无发光特效**）：

| 档位 | 占比 | 内容 |
|-|-|-|
| 常见 | 60% | L1/L2 普通生物、基础装饰、少量光合币 |
| 稀有 | 30% | L3 鱼类、特色装饰、中量光合币 |
| 限定 | 10% | L4 大鱼、节气/天气专属物种、纪念装饰 |

## 5.4 图鉴系统（简化）

- 120 格图鉴，按食物链层级分页（L1–L4 + 访客 + 装饰）
- 仅"**已收藏 / 未收藏**"两态，无稀有度颜色标识
- 每只解锁后配一段冷知识 / 科普卡
- 集齐分类奖励：水生 30 / 植物 40 / 昆虫 30 / 访客 20

## 5.5 反沉迷与节奏

<callout emoji="⏰">
每日事件总数有上限（默认 3 次），避免事件刷屏破坏治愈节奏。事件按"**稀缺感**"设计，让玩家期待"明天的海鸥又会带什么来"。
</callout>

# 7. 技术方案

## 7.1 软件栈

- **框架**：ESP-IDF 5.x
- **UI**：LVGL 9.x（适配 QSPI AMOLED）
- **存储**：NVS（玩家数据） + LittleFS（资源） + SD（截图）
- **音频**：ES8311 I²S 驱动（PCM/MP3）
- **传感**：QMI8658 I²C 驱动 + 步数算法
- **时钟**：PCF85063 + SNTP 校时
- **网络**：Wi-Fi STA + HTTPS 拉取天气 API

## 7.2 数据模型

```c
typedef struct {
    float sun;            // 0-100
    float nutrient;       // 0-100

typedef struct {
```

# 9. 风险与应对

| 风险 | 应对 |
|-|-|
| AMOLED 烧屏 | 桌面伴侣每 60s 像素轻微偏移；定时反色 |
| RAM 不足以承载 LVGL 双缓冲 | 启用 PSRAM 帧缓冲 + 局部刷新 |
| 离线时间被恶意修改 | SNTP 校时 + 硬上限保护 |
| 随机事件过密破坏节奏 | 每日事件上限 3 次 + 冷却时间 |
| 生态计算导致掉帧 | AI 分层（远处低频）+ 生物 ≤ 24 上限 |
| 用户长期不充电体验差 | 低电量切到极简模式仍能运行 |
| 用户对"无 SSR / 无发光"觉得平淡 | 用**事件叙事感**和**鱼会变大**的成长视觉补偿 |

### 11.4.2 Creature 字段新增

```
typedef struct {
    uint16_t species_id;
    uint8_t  trophic_level;   // 1-4
```

### 11.4.3 事件 precondition 字段新增

events.csv 新增列 `precondition`（默认空）。表达式语法：DSL 形式，支持 `&& / || / == / >= / <=`，可引用 GameSave / Creature / 环境字段。

| 长尾 | 触发方式 |
|-|-|
| 28 极乐鸟鱼 | **非事件**，由心跳每日检查 `play_days_total >= 100` 时直接 unlock_codex(28) + 入缸（产 1 条） |
| 29 龙鱼成体 | **非事件**，由进化逻辑（§11.4.2）触发，无需 precondition |
| 30 虎纹鸭嘴 | **挂事件** E005 thunder_night，新增 precondition：`tank_level==10 && rand() < 0.30`，命中时奖励池追加 species_id=30 |

**events.csv 增量**（仅 E005 受影响）：

```
E005,thunder_night,...,reward_pool=creature_l4_rare;creature_limited;decor_dark;creature_id_30,reward_weight=55;25;20;0,...,precondition=null
# 当 GameSave.tank_level==10 时，引擎运行期把权重 [55,25,20,0] 调整为 [40,20,15,25]，让 30 虎纹鸭嘴有 25% 命中
```

<callout emoji="📌">
**实现要点**：precondition 不要硬编码进 CSV 的奖励权重列；保持 CSV 单一权重事实源，由 `event_runtime.cc` 的钩子函数 `resolve_reward_weights(event, save)` 在运行期根据条件动态调整。
</callout>
